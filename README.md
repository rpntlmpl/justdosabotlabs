# РГР: Робота із версійністю баз даних і презентація фінального проєкту

## Виконані завдання:
1.  **Встановлення:** Liquibase встановлено вручну в директорію `/opt/liquibase`.
2.  **База даних:** Використано систему керування базами даних **H2** (вбудований режим).
3.  **Конфігурація:** Створено файл `liquibase.properties` для автоматизації підключення до БД.
4.  **Міграції:** Розроблено журнал змін `changelog.xml` з двома наборами змін (changesets):
    * `id: 1` — створення таблиці `users` (id, username).
    * `id: 2` — додавання колонки `email` до таблиці `users`.
5.  **Тестування:** Успішно перевірено життєвий цикл міграцій за допомогою команд `status`, `update` та `rollback`.

## Запуск проекту локально:

Для роботи з міграціями необхідно мати встановлену Java (JRE).

1.  **Перевірка стану:**
    Дізнатися про наявність не застосованих змін:

    liquibase status
   

2.  **Застосування міграцій:**
    Оновити схему бази даних до останньої версії:

    liquibase update
 

3.  **Відкат змін:**
    Повернути базу даних на один крок назад:

    liquibase rollbackCount 1
  

## Структура файлів проекту:
* `index.html` — головна сторінка веб-сайту.
* `Dockerfile` — інструкції для збірки Docker-образу на базі Ubuntu 24.04.
* `.github/workflows/` — конфігурація CI/CD пайплайну.
* `load-tests/` — сценарії навантажувального тестування за допомогою K6.
* `changelog.xml` — журнал усіх змін структури БД.
* `liquibase.properties` — налаштування з'єднання з базою даних.
* `h2-2.2.224.jar` — JDBC-драйвер для роботи з БД H2.
